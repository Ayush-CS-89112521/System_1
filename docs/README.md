# Documentation Guide

This documentation is organized from project intent to implementation and operations.

## Recommended Reading Path

1. Start at workspace overview:
- `../README.md`

2. Understand project goals and baseline setup:
- `00-overview/CONSOLIDATION_REPORT.md`
- `00-overview/SETUP_MATRIX.md`

3. Understand architecture boundaries:
- `01-architecture/ARCHITECTURE_MAP.md`

4. Review governance and security model:
- `02-governance/SECURITY_BASELINE.md`
- `02-governance/ENV_AND_AUTH_MATRIX.md`
- `02-governance/CONTRIBUTOR_WORKFLOW.md`

5. Move into implementation details:
- `03-implementation/WATCHDOG_PROTOCOL.md`
- `03-implementation/SAFETY_PROXY.md`

6. Run and operate safely:
- `04-operations/OPERATIONS_RUNBOOK.md`

7. Track implementation progress:
- `05-planning/MIGRATION_TRACKER.md`

## Folder Structure

- `00-overview`: strategy and setup context
- `01-architecture`: system boundaries and topology
- `02-governance`: policy, auth boundaries, contributor rules
- `03-implementation`: technical guardrails and component contracts
- `04-operations`: daily runbook and emergency procedures
- `05-planning`: migration and execution tracking

## Documentation Rule

Workspace-level policy and process docs stay in this folder structure.
Project-specific implementation docs stay inside each project repository.
